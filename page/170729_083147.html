  • memtest86.

    memtest86 (not memtest86+) is utility to do memory test on x86 or x86_64 (amd64) PC. This utility seems as same as same package name in Debian repository, but I installed this utility to usb by downloading the tar.gz file from website instead of install with aptitude (maybe will install on OS and use grub to load it when boot, I don't konw).

    Installing is quite simple, just download tar.gz file from website, follow the README file use dd command to write it to usb, nothing more. This utility have free version and pro version, there is not much different so just download free version. And later when you are runing free version the interface will offer option to run v7.x or v4.x. v4.x version seems design for 32bit hardware, and you can't test memory more than 4GB.

    Make sure you disable UEFI compatible mode (this will cause disk or OS using MBR partition table not able to show in boot menu too), boot PC with this USB stick plugin and the utility will able to run. Select v7.x version instead of v4.x.

  • Installing Chinese input method.

    • Install ibus-chewing (酷音) package first.
    • Type ibus-setup command, go to second tab (Input Method), add newly installed ibus-chewing input method. (If you still don't see ibus-chewing after install, you probably need to reboot your system).
    • Then follow Gnome Help tool, [search / find] Use alternative keyboard layouts section and follow the steps, that is, open Region & Language and enable ibus-chewing to join the input sources switch (after this, you can see the effect at layout indicator at top bar). The defualt key to switch different input method is Super + space.

  • [dconf // gsettings].

    • What is [dconf // gsettings] : Here - dconf is a low-level configuration system. Its main purpose is to provide a backend to GSettings on platforms that don't already have configuration storage systems.

      gsettings is a command line tool for viewing and modifying GSettings, and works regardless of the GSettings backend in use. For modifying the dconf backend storage itself, use the dconf tool; but gsettings should be used by preference.

      Here - GSettings are a nice way to manage your application’s settings. Those settings are stored in binary form for fast retrieval, and can be manipulated outside of your application as well using specific tools. This system can be seen as a database, whose schema is described in an XML file (conventionally named .gschema.xml) ...

      • keyboard repeat rate : See here. Just set like this and now keybord delay and repeat-interval of keyboard is nice

        gsettings set org.gnome.desktop.peripherals.keyboard delay '300' (default = 500. uinit32 is probably data type, ignore it.)
        gsettings set org.gnome.desktop.peripherals.keyboard repeat-interval '23' (default = 30)
