coreNote related

  • About coreNote : Start at here if you don't know my coreNote html template. (Click below button to expand).

    • Why?

      I build coreNote mainly to get some important information when my PC is accidentally not accessible, for example during OS installation, my [local CMS // database // asKansweR password script] is all not accessible, and cause all my information lockdown while some of them are important.

      When network is probably accessible through smartphone, I can still read those HTML document. If my PC is able to use browser, I can use either firefox or w3m to navigate those HTML document structure. (Aware, w3m can't rander iframe properly, but you can still read most basic information).

      coreNote keeps minimal neccessary information still accessible anytime.

    • [Feature / function].

      I still no much time to document this well, but this is simple introduction about [Feature / function].

      • Iframe generate duplicate content.

        When I have a piece of information I will used again and again, I don't have to [copy // paste // midify] it every place where this piece of information is, instead, I just need to create another special HTML page, and anytime I need this piece I use iframe tag to call it. The most benifit about this "iframe calling" is, modify one source place, apply to all destination at once, never worry about duplicate or accidentally typing error.

        PHP have a common used feature called include, it is the same idea in compare. Though "iframe calling" is just simple javascript it doesn't need preprocessor, it works on static site like github, and it works simply.

        In fact, in top-right side of every page of coreNote, you always see coreNote text, that piece of text is generate by iframe, not wrote inside each HTML document page (view source code of this page to see it).

      • Main section block [expand // collapse] : At left side of screen, the vertical bar is actually a hidden button, click it to toggle between [expand // collapse] each section block.

      • "code" block [expand // collapse] : You can put anything inside "code" block not only code, when put any information inside code block, it is able to toggle between [expand // collapse], thus save space if some information is not important or too long.

      • Side menu (Top-right menu) : Top-right side of the screen, the menu include some features (buttons)

        • Jump to head section.
        • Expand mode : Expand all code block in document.
        • Compact mode : Reset the [expand // collapse] of all code block in document to it's default value. _ When I was writing the document, I will hide some code block (just simply add hide class name on code block), browser will hide those code block automatically, and left others untouched.
        • Minimal mode : Minimize (collapse) all code block in document. _ When reading document with mobile device, this might be very useful to navigate through the long document with many ul, li structure because the screen is small and limited. Collapse all code block first than open it if needed.

      • Hidden "jumpto" menu : Left side of the docuemnt, you can see some triangle strips, those are actually hidden button. Click on those triangle "button" to pop out advenced menu.

        • Triangle strips : To visually indicate the percentage of scrolling position. Because it's triangle shape, you can guess how much percentage you where reading in each li block, when the strip is thinner, you know you are more close to reach the end (100%) of each li block.

          When reading document with mobile device, this might be very useful to navigate through the long document with many ul, li structure because the screen is small and limited, those triangle visual indicator give you idea of where you are reading.

          Also this is useful when writing HTML document. When using a lot ul, li structure, I often get lost when writing them because the HTML markup is not as easy to read as in browser (randered).

        • Quit : Quit menu (cross button).
        • Back : Jump to last move. When using jump to bookmark, jump to li top, jump to li bottom buttons, the previus location will be remembered (one step only), if you "regret" the lattest move, use this button to jump back previous screen position.

        • jump to li top, jump to li bottom.

          Jump to selected li top. coreNote use ul, li a lot to structure document in list order. When you get lost in long ul, li structure, you can use specific triangle strip you want, click on it brings up the menu, then click jump to li top or jump to li bottom buttons bring you to the most [top / bottom] of this selected li.

          This can navigate back current li heading or to next li heading very quick.

          Note the "destination" will placeed at the center of the screen, not the top of screen.

        • Bookmarks : You can save 3 bookmarks at max, use + button to save current screen position, and use "precentage button" to jump to that bookmark position.

    • Some problem.

      • Not optimise for mobile device, CSS is not coding media query yet, so smart phone will have some glitch style (for example the bookmark only shows 2 instead of 3 because flexbox is overflow).
      • Bookmark in Hidden "jumpto" menu will lose accuracy after you change between [expand // compact// minimal] mode, because bookmark only save position (distance) between viewport and document top.
